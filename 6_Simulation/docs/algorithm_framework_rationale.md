# 控制算法与仿真框架选型说明

## 1. 本章目的

本文仿真系统的核心任务是在有限视场条件下，使追踪无人机稳定、平滑地接近运动目标，并尽可能减少目标离开视场的时间。`6_Simulation` 项目围绕这一任务建立了两级验证链路：离线 Python 三维质点仿真用于公平比较算法本身，ROS2/PX4/Gazebo 双机闭环仿真用于验证算法接入飞控仿真链路后的可执行性。

在该设计中，本文选择 `PN + FOV + NMPC` 作为重点控制算法，选择 PX4/ROS2 作为闭环仿真框架。二者的选择并不是为了追求捕获时间或最小距离等单一指标最优，而是为了匹配有限视场目标追踪问题的关键矛盾：既要接近目标，又要让目标持续保持在机头或相机视场内，同时还要保证控制输入和 yaw 运动不过于剧烈。

## 2. 问题特性决定算法需要分层设计

有限视场目标追踪与理想传感器条件下的追踪不同。若假设目标始终可见，导引算法只需要根据目标当前位置和速度生成拦截轨迹；但在实际视觉追踪中，目标一旦离开相机视场，导引输入就会退化为最后一次观测及其短时预测。此时，单纯追求快速接近目标反而可能导致近距离飞越、LOS 方向快速变化和长时间失锁。

项目中的模型和指标也体现了这一点。追踪机不是完整六自由度气动模型，而是带速度、加速度、高度、yaw/pitch 角速度约束的三维质点模型；yaw/pitch 主要用于表达机头或相机朝向。评价指标不仅包括捕获时间和最小距离，还包括 `visible_ratio`、`lost_duration`、`control_energy`、`yaw_rate_mean` 和 `yaw_rate_variance`。因此，合适的算法不能只解决“追上目标”，还必须同时处理“看住目标”和“平滑执行”。

基于这一问题结构，`PN + FOV + NMPC` 采用分层组合：PN 提供稳定、可解释的拦截趋势；FOV 模型把视觉可见性引入导引输入；NMPC 在短预测窗口内对距离、视场和控制平滑性进行折中优化。

## 3. 为什么使用 PN 作为导引基础

PN（Proportional Navigation，比例导引）适合作为本文算法的底层名义导引律，主要原因有三点。

第一，PN 是面向拦截问题的经典几何导引方法。相比 Direct pursuit 始终指向目标当前位置，PN 利用相对位置、相对速度和 LOS 角速度生成横向加速度，更适合处理匀速或机动目标。项目中的 `pn_guidance()` 采用三维向量形式计算 LOS 角速度，并加入径向闭合项，使追踪机在初始速度较小时也能主动接近目标。

第二，PN 的输出具有明确物理含义，便于作为更复杂控制器的名义趋势。本文的 NMPC 和 MPPI 都不是从零搜索任意控制，而是以 PN 加速度为中心构造候选或名义控制序列。这种做法限制了搜索空间，避免预测控制在短窗口内选出背离拦截任务的异常控制，同时保留了 PN 的工程可解释性。

第三，PN 计算量低、参数少，适合作为实时导引外环。项目的 Gazebo 节点以固定控制频率读取两机 `VehicleOdometry` 并发布 `TrajectorySetpoint`，导引计算需要在每个控制周期稳定完成。PN 可以为实时链路提供可靠基础，而预测控制只在此基础上做有限范围的增强。

## 4. 为什么必须引入 FOV 约束

若只使用 PN，算法默认目标状态始终可用，这与视觉追踪任务不一致。有限视场约束的引入有两个作用。

第一，它把传感器可见性从事后评价变成控制过程的一部分。项目中通过追踪机 yaw/pitch 得到前向向量，再计算目标 LOS 与前向向量之间的夹角。当夹角超过 FOV 半角时，目标被判定为不可见。此时 FOV 算法不继续“偷看”真实目标，而是使用最后一次观测到的位置和速度做常速度外推，并降低导引增益。这一机制模拟了视觉目标短时离开画面后的追踪退化过程。

第二，它暴露了传统导引在视觉任务中的不足。离线结果显示，`PN + FOV` 在三种场景中都能完成接近，但会出现较长失锁时间。例如静止、直线和圆周场景中的 `PN + FOV` 可见率分别约为 0.246、0.385 和 0.375；Gazebo 圆周场景中其可见率也只有 0.615。说明单纯在 PN 外层加入 last-seen 预测并不足以稳定保持目标在视场内，尤其在目标横向机动和近距离交会时更明显。

因此，FOV 不是一个附加绘图指标，而是本文控制问题的核心约束。它使算法评价从理想拦截转向更接近视觉追踪的闭环行为。

## 5. 为什么在 PN 和 FOV 基础上加入 NMPC

有限视场追踪存在多目标折中：快速接近目标需要较强机动，保持目标在视场中心又要求 LOS 变化不过快，控制平滑性还要求加速度和 yaw 变化不能过于剧烈。PN 本身不显式处理这些折中，CBF 风格修正主要关注 FOV 边界附近的即时安全，MPPI 能通过采样改善视场保持并保持较快捕获，但计算成本、随机采样参数和控制平滑性压力更高。NMPC 的优势在于可以在预测窗口中统一评价这些目标。

本文实现的 `PN + FOV + NMPC` 是轻量级 shooting-based candidate optimization，而不是依赖 CasADi、acados 或 Ipopt 的完整非线性规划求解器。其流程为：先计算 PN 名义加速度，再构造有限个候选加速度，包括缩放 PN、直接拦截、同速接近、速度匹配以及 LOS 垂直方向扰动；随后在 2.0 s 预测窗口内滚动仿真每个候选，使用终端距离、路径距离、FOV 违反、控制能量、控制平滑性和偏离 PN 趋势等代价进行评分，最终只执行当前周期代价最低的候选。

这种设计适合本项目有五个原因。

第一，它保留了 PN 的可解释拦截趋势。代价函数中包含偏离 PN 趋势的惩罚，候选集合也围绕 PN 构造，因此 NMPC 是 PN 的预测增强层，而不是完全独立的黑箱控制器。

第二，它能显式权衡 FOV 和捕获。FOV 违反项会惩罚预测窗口内超出半视场角的行为，终端距离和路径距离仍推动追踪机接近目标。算法因此能够在“立即抄近路捕获”和“稍微保守以保持可见”之间做滚动选择。

第三，它能降低控制强度和 yaw 抖动。控制能量和平滑项会惩罚过大的加速度和相邻周期控制突变。根据当前 `outputs` 结果，NMPC 在静止、直线、圆周三种离线场景中的控制能量分别为 62.49、53.58 和 124.97，均为各算法最低；yaw rate mean 分别为 0.015、0.070 和 0.226 rad/s，同样最低。这说明当前参数下 NMPC 明显偏向低能耗、低 yaw 速率和视场稳定，而不是激进拦截。

第四，它能显著提高有限视场保持能力。当前离线结果中，NMPC 在三种场景的 `visible_ratio` 均为 1.000，`lost_duration` 均为 0.00 s；相比之下，`PN + FOV` 在静止、直线、圆周场景中的可见率分别约为 0.246、0.385 和 0.375，说明单纯 last-seen 预测无法主动避免长时间失锁。

第五，它比完整 NMPC 求解器更轻量，便于和 PX4/ROS2 实时链路集成。候选数量有限，动力学模型与离线仿真一致，计算过程不依赖外部优化器，降低了环境配置和实时失败风险。对于本文的验证目标而言，这种轻量滚动优化已经可以体现“预测视场保持”的核心思想。

需要注意的是，NMPC 并不保证所有指标都绝对最优。当前离线结果中，NMPC 在静止、直线、圆周场景的捕获时间分别为 7.55 s、6.15 s 和 6.30 s，均不如 MPPI 激进；最小距离也不如 PN 或 MPPI。Gazebo 圆周场景中，NMPC 的可见率最高、丢失时长最低、yaw 最平滑，但捕获时间长于 MPPI。这说明该算法更偏向视场保持、低控制能量和平滑控制，而不是单纯最短时间捕获。这一取舍与有限视场视觉追踪任务的需求一致。

## 6. 为什么不只使用其他对比算法

Direct pursuit 适合作为直观基线，但它只追踪目标当前位置，不利用 LOS 角速度和目标速度预测。面对横向运动或圆周机动目标时，它容易产生较弯轨迹，拦截效率低。

3D PN 能显著改善拦截效率，但默认目标始终可见。若直接用于视觉目标追踪，它不能解释目标离开视场后的信息退化，也不会主动约束 LOS 留在相机视场内。

`PN + FOV` 引入了有限视场和 last-seen 预测，是必要基线，但结果表明它只能被动处理目标丢失，不能充分提前规避失锁。

`PN + FOV + CBF` 能在接近 FOV 边界时施加侧向修正，但当前项目实现是 CBF 风格的启发式安全滤波器，没有求解严格 CBF-QP。它适合做局部边界修正，但对复杂机动和多目标折中改善有限。

`PN + FOV + MPPI` 通过随机采样可以显著提高可见率，并且在当前离线结果中通常保持更快捕获；但其控制能量和 yaw 平滑性不如 NMPC。MPPI 还引入采样数、噪声尺度、温度和随机种子等额外因素。相比之下，NMPC 候选由导引几何和任务意图构造，更容易解释每类候选的作用。

因此，本文选择 `PN + FOV + NMPC` 作为重点算法，是因为它在可解释性、有限视场保持、控制平滑性和实时集成复杂度之间取得了更合适的平衡。该选择的核心不是证明 NMPC 在捕获时间上最优，而是证明其能在有限视场约束下实现更稳定的目标可见性和更平滑的控制行为。

## 7. 为什么使用 PX4/ROS2/Gazebo 框架

离线 Python 仿真可以公平比较算法，但它不能回答算法接入真实飞控接口后是否还能闭环运行的问题。PX4/ROS2/Gazebo 框架的作用正是补上这一层验证。

第一，PX4 提供接近真实无人机系统的飞控闭环。项目中的 ROS2 节点并不直接模拟电机、姿态或底层控制器，而是通过 PX4 Offboard 模式发布速度和加速度 setpoint。这样可以把导引算法限定在外部任务层，让姿态稳定、位置/速度跟踪和飞控状态机由 PX4 处理。相比纯 Python 质点积分，PX4 会引入底层控制滞后、setpoint 跟踪误差和模式切换流程，因此更能检验算法在飞控闭环中的可执行性。

第二，ROS2 是连接算法、PX4 消息和数据记录的合适中间层。项目中的 `guidance_node.py` 订阅两架无人机的 `VehicleOdometry`，发布 `OffboardControlMode`、`TrajectorySetpoint` 和 `VehicleCommand`；`px4_msgs` 提供 uORB 消息在 ROS2 中的接口定义。ROS2 的命名空间机制也适合双机仿真，默认 `/px4_1` 为追踪机、`/px4_2` 为目标机，算法、场景、控制频率和 namespace 都可通过 launch 参数覆盖。

第三，Gazebo 允许在同一仿真世界中构建目标机和追踪机的闭环交互。目标机跟随 `target_state()` 生成的参考轨迹，但追踪机导引输入使用目标机真实 PX4 odometry，而不是理想目标参考。因此，目标机跟踪误差会进入追踪机的闭环导引输入，使验证比离线理想目标轨迹更严格。

第四，该框架保持了离线算法和闭环接入的一致性。导引核心仍调用 `pythonsimulation.guidance.compute_guidance()`，Gazebo 节点只负责坐标转换、Offboard setpoint 发布、启动阶段管理和数据记录。算法内部统一使用 ENU 坐标系，PX4 接口边界通过 `coordinates.py` 转换为 NED 坐标系。这样避免维护两套导引实现，降低离线结果和 Gazebo 结果不一致的风险。

第五，PX4/ROS2 数据链路便于形成可复现实验结果。节点退出时保存 `gazebo_samples.csv`，字段包括两机位置速度、加速度指令、yaw、pitch、距离、可见性和 LOS 角；后处理脚本 `plot_gazebo_csv.py` 复用离线仿真的指标和绘图函数。这样离线仿真和 Gazebo 闭环仿真可以使用同一套指标体系进行对照。

## 8. 离线仿真与 PX4/ROS2 验证的关系

离线仿真和 PX4/ROS2/Gazebo 仿真承担不同角色，不能互相替代。

离线 Python 仿真用于算法层公平比较。所有算法共享相同初始状态、目标轨迹、速度/加速度限制、FOV 参数和评价指标，适合分析算法设计本身是否有效。当前离线结果显示，NMPC 在三种场景中均实现全程可见（`visible_ratio=1.000`，`lost_duration=0.00 s`），同时控制能量和 yaw rate mean 均为最低，说明预测视场代价和控制平滑代价是有效的；但其捕获时间和最小距离不是最优，说明该参数组采用的是保守、平滑的追踪策略。

PX4/ROS2/Gazebo 仿真用于系统层闭环验证。它引入 PX4 Offboard 模式、真实消息接口、双机 odometry、底层控制滞后和坐标系转换，适合检验算法能否通过飞控仿真链路运行。Gazebo 结果与离线仿真趋势不完全一致，例如 NMPC 在圆周场景中捕获时间变长，但它仍在 FOV 约束算法中取得最高可见率和最低丢失时长。这说明 PX4 闭环会改变捕获性能，但算法的视场保持优势仍然存在。

因此，本文采用两级验证而不是只使用其中之一：离线仿真证明算法思想，PX4/ROS2/Gazebo 验证工程接入和闭环可执行性。

## 9. 适用边界与局限

本文选择 `PN + FOV + NMPC` 和 PX4/ROS2 框架，并不意味着当前仿真已经覆盖全部真实飞行因素。离线部分仍是三维质点模型，yaw/pitch 只用于有限视场建模，不包含完整四旋翼姿态、推力、电机和气动模型。FOV 可见性只考虑几何夹角，没有模拟图像检测噪声、遮挡、延迟和误检漏检。目标丢失后的预测采用常速度模型，不能覆盖强机动目标的复杂行为。

PX4/Gazebo 部分虽然接入了飞控闭环，但结果仍会受到 PX4 控制器参数、机体模型、目标机跟踪误差、通信频率和 Offboard setpoint 形式影响。因此，本文结论应表述为：该算法在当前有限视场追踪仿真设定下表现出较好的视场保持和控制平滑性，并已通过 PX4 Offboard 闭环仿真验证其可执行性；后续若面向实机，还需要进一步加入真实视觉检测、状态估计误差、通信延迟和机体动力学约束。

## 10. 小结

选择 `PN + FOV + NMPC` 的核心理由是：PN 提供经典、稳定、可解释的三维拦截趋势；FOV 模型把视觉传感器可见性纳入导引输入和评价指标；NMPC 通过短时滚动预测在捕获、视场保持、控制能量和平滑性之间做显式折中。当前仿真结果表明，该组合特别适合有限视场目标追踪中“看住目标”与“平滑接近目标”的综合需求，但不应被表述为最短时间捕获算法。

选择 PX4/ROS2/Gazebo 的核心理由是：PX4 提供标准 Offboard 飞控接口和底层闭环，ROS2 提供多机消息通信、参数化启动和数据记录，Gazebo 提供双机仿真环境。该框架使同一套 Python 导引算法可以从离线公平对比平滑过渡到飞控闭环验证，从而支撑本文算法不仅在数值模型中有效，也能在更接近无人机系统的软件链路中运行。
