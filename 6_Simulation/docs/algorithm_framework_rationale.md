# 仿真中 PN + FOV + NMPC 算法框架原理分析

## 1. 框架目的

本章说明仿真系统采用 `PN + FOV + NMPC` 的核心原因。该框架面向有限视场条件下的无人机目标追踪：追踪机既要接近运动目标，又要尽量保持目标位于相机视场内，并在速度、加速度、高度和机头转向速率受限的条件下输出平滑控制。

三部分分工如下：

1. **PN** 提供基于相对运动几何的名义拦截趋势；
2. **FOV** 将目标可见性、目标丢失和状态估计引入导引闭环；
3. **NMPC** 在短预测窗口内权衡目标接近、视场保持和控制平滑性。

因此，该框架的目标不是单纯最快接近，而是在“接近目标、保持可见、控制可执行”之间取得平衡。

## 2. PN：提供名义拦截趋势

PN（比例导引）的作用是根据追踪机与目标的相对位置、相对速度和视线角速度生成三维拦截指令。设追踪机位置为 `p_p`，目标位置为 `p_t`，则相对位置和单位视线方向为：

```text
r = p_t - p_p
u_LOS = r / ||r||
```

相对速度、闭合速度和视线角速度为：

```text
v_rel = v_t - v_p
V_c = max(0, -v_rel · u_LOS)
omega_LOS = (r × v_rel) / ||r||^2
```

PN 横向加速度为：

```text
a_PN = N V_c (omega_LOS × u_LOS)
```

该项用于抑制 LOS 旋转，使追踪机向预期交会方向调整。为了增强沿视线方向的接近能力，可加入径向闭合项：

```text
a_close = k_close (v_des - v_p · u_LOS) u_LOS
a_nom = a_PN + a_close
```

在组合框架中，PN 不负责处理全部约束，而是给出物理意义明确的名义控制方向。NMPC 后续围绕该趋势进行有限修正，避免控制完全偏离拦截几何。

## 3. FOV：约束可见性与目标参考

有限视场追踪不能假设目标状态始终可用。目标在视场内时，可以利用量测更新目标估计；目标离开视场后，只能依靠已有估计做短时预测。因此，FOV 是导引闭环的一部分，而不只是事后评价指标。

设机头或相机前向向量为 `f`，目标方向为 `u_LOS`，两者夹角为：

```text
alpha = arccos(clip(f · u_LOS, -1, 1))
```

若半视场角为 `phi`，则目标可见条件为：

```text
visible = (alpha <= phi)
```

目标可见时，量测位置进入 α-β 滤波，更新目标位置和速度估计：

```text
p_pred = p_hat + v_hat dt
r = z_p - p_pred
p_hat = p_pred + alpha_f r
v_hat = v_hat + beta_f r / dt
```

目标不可见时，不再使用真实目标状态，而采用常速度预测：

```text
p_hat = p_hat + v_hat dt
```

FOV 同时影响朝向控制：当前可用的目标参考位置会作为机头或相机的指向参考，使系统持续尝试将目标保持在视场内。目标短时丢失时，导引指令应更保守，以反映观测不确定性增加。

## 4. NMPC：短时预测下的控制选择

PN 能给出拦截趋势，FOV 能描述目标是否可见，但当前控制仍可能在未来几步导致目标出视场、控制突变或近距离快速飞越。NMPC 的作用是在执行当前控制前，先预测短时未来并比较候选控制的综合代价。

本框架中的 NMPC 可视为轻量候选式预测控制。候选控制围绕 PN 名义加速度构造，主要包括：

1. 不同强度的 PN 控制；
2. 更强调沿 LOS 接近的控制；
3. 更强调速度匹配的控制；
4. LOS 垂直方向上的小幅调整。

这些候选既保留 PN 的拦截趋势，又允许控制器为了保持视场和改善平滑性进行有限修正。

对每个候选控制，NMPC 在预测窗口内模拟追踪机和目标的相对运动，并计算总代价：

```text
J =
    w_dist    J_dist
  + w_path    J_path
  + w_fov     J_fov
  + w_control J_control
  + w_smooth  J_smooth
  + w_PN      J_PN
```

其中，`J_dist` 表示预测末端距离，`J_path` 表示预测过程距离，`J_control` 抑制过大加速度，`J_smooth` 抑制控制突变，`J_PN` 限制控制过度偏离 PN 趋势。FOV 违反项为：

```text
J_fov = sum_i max(0, alpha_i - phi)^2
```

当 LOS 位于视场内时，该项为零；当 LOS 超出半视场角时，惩罚随超出角度增大而增加。这样，NMPC 会优先选择既能接近目标、又不易导致目标出视场的当前控制量。

## 5. 闭环流程与小结

每个控制周期中，三者的闭环关系可概括为：

1. 根据 FOV 可见性生成当前目标参考状态；
2. PN 根据目标参考状态计算名义拦截指令；
3. NMPC 围绕 PN 指令生成候选控制并进行短时预测；
4. 根据距离、FOV、控制能量、平滑性和 PN 偏离代价选择当前控制量；
5. 根据目标参考位置更新机头或相机朝向；
6. 下一周期利用新的状态和量测重新计算。

综上，`PN + FOV + NMPC` 构成了“名义拦截 + 可见性约束 + 短时预测优化”的导引链路。PN 提供基础拦截能力，FOV 使目标可见性和估计状态进入闭环，NMPC 则在短时未来中权衡距离、视场和控制品质，从而更适合有限视场目标追踪仿真。
