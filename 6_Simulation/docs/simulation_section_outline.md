# 论文仿真部分详细大纲

## 1. 仿真目的与验证思路

本节用于说明仿真章节的总体目标。建议开篇明确：本文提出的重点算法为 `PN + FOV + NMPC`，其余算法作为对比基线，用于验证该方法在目标追踪、视场保持和控制平滑性方面的综合性能。

可写要点：

- 仿真目的不是建立高保真四旋翼气动模型，而是验证导引与视场约束处理方法的有效性。
- 仿真采用两级验证框架：离线 Python 三维质点仿真和 ROS2/PX4/Gazebo 双机闭环仿真。
- 离线仿真用于在严格相同初始条件、目标轨迹和物理约束下进行多算法横向对比。
- Gazebo 闭环仿真用于验证算法通过 PX4 Offboard 接口接入实际飞控仿真链路后的可执行性。
- 本文重点关注的性能包括捕获时间、最小距离、目标可见率、视场丢失时长、控制能量和 yaw 角速度平滑性。

建议正文表达：

> 为验证有限视场条件下无人机追踪导引方法的有效性，本文构建了离线数值仿真与 PX4/Gazebo 闭环仿真相结合的验证框架。离线仿真用于公平比较不同导引算法的理论性能，Gazebo 仿真用于进一步检验算法在 ROS2 与 PX4 Offboard 控制链路中的闭环运行效果。

## 2. 仿真系统总体结构

本节介绍 `6_Simulation/src/` 的代码组织对应的仿真架构。

### 2.1 离线 Python 仿真模块

对应源码：`src/pythonsimulation/`。

可写要点：

- `config.py` 定义算法名称、目标场景和统一仿真参数。
- `state.py` 定义追踪机状态、目标状态和仿真结果数据结构。
- `target.py` 生成静止、直线和圆周三类目标轨迹。
- `dynamics.py` 实现追踪机三维质点运动模型和 yaw/pitch 朝向更新。
- `guidance.py` 实现 Direct pursuit、Direct pursuit + FOV、PN + FOV、PN + FOV + CBF、PN + FOV + MPPI、PN + FOV + NMPC。
- `simulation.py` 负责统一仿真循环，保证各算法在相同目标轨迹、初始条件和物理约束下运行。
- `metrics.py` 和 `plotting.py` 分别负责性能指标计算与图表生成。
- `replay.py` 可导出 Foxglove/MCAP 回放文件，用于检查三维追踪过程。

### 2.2 ROS2/PX4/Gazebo 闭环仿真模块

对应源码：`src/gazebosimulation/` 与 `src/px4_msgs/`。

可写要点：

- `gazebosimulation` 是 ROS2 Python 包，用于把离线导引算法接入 PX4/Gazebo 双机仿真。
- `guidance_node.py` 控制两架 PX4 实例，默认 `/px4_1` 为追踪机，`/px4_2` 为目标机。
- 节点订阅两机 `VehicleOdometry`，发布 `OffboardControlMode`、`TrajectorySetpoint` 和 `VehicleCommand`。
- 目标机跟随 `target_state()` 生成的参考轨迹，追踪机则使用目标机真实 PX4 odometry 作为闭环导引输入。
- `px4_msgs` 提供 PX4 uORB 消息在 ROS2 中的接口定义，是 PX4 与 ROS2 通信的消息依赖。
- Gazebo 仿真结果保存为 `outputs/gazebo/<scenario>/<algorithm>/gazebo_samples.csv`，再由 `plot_gazebo_csv.py` 后处理生成与离线仿真一致的指标和图表。

## 3. 坐标系与状态变量定义

本节建议先统一符号，避免后续描述中 ENU/NED 混淆。

### 3.1 离线仿真坐标系

可写要点：

- 离线算法统一使用 ENU 坐标系。
- 坐标轴定义为 `x` 指向 East，`y` 指向 North，`z` 指向 Up。
- 追踪机状态包括位置 `p_p`、速度 `v_p`、加速度 `a_p`、yaw 和 pitch。
- 目标状态包括位置 `p_t`、速度 `v_t`、加速度 `a_t`。
- yaw 表示水平朝向，yaw=0 表示朝 ENU 的 `+x` 方向。
- pitch 表示俯仰角，pitch=0 表示水平向前，正值表示向上。

### 3.2 Gazebo/PX4 坐标转换

可写要点：

- PX4 的 `VehicleOdometry` 和 `TrajectorySetpoint` 使用 NED 坐标系。
- 算法内部仍保持 ENU 坐标系，只有 ROS2/PX4 接口边界执行坐标转换。
- 位置、速度和加速度统一使用转换关系 `[x_E, y_N, z_U] -> [y_N, x_E, -z_U]`。
- yaw 角通过 `yaw_ned = pi/2 - yaw_enu` 转换。
- 这种设计避免维护两套算法实现，降低 Gazebo 接入与离线仿真之间的不一致风险。

## 4. 追踪机运动模型与物理约束

本节介绍离线仿真的三维质点模型，也是所有对比算法共享的执行约束。

### 4.1 三维质点模型

可写要点：

- 导引算法输出加速度指令 `a_cmd`。
- 执行前对加速度进行幅值限幅，保证 `||a_cmd|| <= a_max`。
- 速度由加速度离散积分得到，位置由速度离散积分得到。
- 速度同样限幅，保证 `||v|| <= v_max`。
- 高度限制在 `[z_min, z_max]` 范围内，触碰高度边界时清除继续越界的垂向速度。

建议公式：

```text
a_k = sat(a_cmd, a_max)
v_{k+1} = sat(v_k + a_k Delta t, v_max)
p_{k+1} = p_k + v_{k+1} Delta t
```

### 4.2 机头/相机朝向模型

可写要点：

- yaw/pitch 不表示完整姿态动力学，而用于建模有限视场方向。
- 每个仿真步中，追踪机朝向会以最大 yaw/pitch 角速度逐渐转向 `look_at_position`。
- 对无 FOV 限制的算法，`look_at_position` 为真实目标位置。
- 对 FOV 算法，目标可见时 `look_at_position` 为深度相机量测经 α-β 滤波后的估计点，目标丢失后为滤波器常速度预测点。

### 4.3 默认仿真参数

本文按代码默认值写入：

| 参数 | 数值 | 含义 |
| --- | ---: | --- |
| `dt` | 0.05 s | 主仿真步长 |
| `sim_time` | 40 s | 单次仿真时长 |
| `capture_radius` | 1.5 m | 捕获判定半径 |
| `v_max` | 12 m/s | 追踪机最大速度 |
| `a_max` | 6 m/s^2 | 追踪机最大加速度 |
| `yaw_rate_max` | 90 deg/s | 最大 yaw 角速度 |
| `pitch_rate_max` | 60 deg/s | 最大 pitch 角速度 |
| `z_min` | 0.5 m | 最低高度 |
| `z_max` | 30 m | 最高高度 |
| `FOV` | 90 deg | 总视场角 |
| `FOV half angle` | 45 deg | 可见性判定半角 |

## 5. 目标运动场景设计

本节说明三种目标轨迹及其作用。

### 5.1 静止目标场景

默认参数：目标固定在 `[40, 20, 8] m`。

可写要点：

- 该场景用于验证算法最基本的收敛能力。
- 对静止目标，Direct pursuit 和 PN + FOV 类算法均应具备基本捕获能力。
- FOV 相关算法在该场景中主要体现视场保持与控制平滑性的差异。

### 5.2 匀速直线目标场景

默认参数：初始位置 `[25, -20, 8] m`，速度 `[2, 1, 0.2] m/s`。

可写要点：

- 该场景用于验证算法是否能够处理目标速度，而不是只追踪当前位置。
- PN 类算法可利用相对速度和 LOS 变化实现更接近拦截的运动。
- 该场景适合比较 Direct pursuit 与 PN + FOV/NMPC 在捕获时间和平均距离上的差异。

### 5.3 圆周机动目标场景

默认参数：圆心 `[35, 0, 8] m`，半径 `12 m`，角速度 `0.25 rad/s`，高度叠加正弦变化。

可写要点：

- 该场景持续改变 LOS 方向，是有限视场追踪中最具挑战性的测试。
- 目标横向机动会导致追踪机更容易出现视场丢失。
- 该场景是突出 `PN + FOV + NMPC` 算法优势的重点分析场景。

## 6. 有限视场可见性模型

本节应服务于本文重点算法，即说明为什么需要在 PN 外层加入 FOV 与 NMPC 约束。

### 6.1 前向向量与 LOS 夹角

可写要点：

- 追踪机前向向量由 yaw 和 pitch 决定。
- 目标相对方向为 `p_t - p_p`。
- LOS 夹角为前向向量与目标相对方向的夹角。
- 当 LOS 夹角小于等于半视场角时，目标被认为可见。

建议公式：

```text
f(yaw, pitch) = [cos(pitch)cos(yaw), cos(pitch)sin(yaw), sin(pitch)]^T
theta_LOS = arccos( f^T (p_t - p_p) / ||p_t - p_p|| )
visible = theta_LOS <= theta_FOV / 2
```

### 6.2 深度相机量测、α-β 滤波与目标丢失预测

可写要点：

- 当目标仍可见时，FOV 算法通过理想深度相机得到目标中心在机体/相机系下的三维点，再转换到地面系位置量测。
- 地面系位置量测进入 α-β 滤波器，估计目标位置和速度，作为 FOV 类算法的目标参考。
- 当目标不可见时，算法不直接使用真实目标状态，而是使用滤波器状态做常速度预测。
- 初始不可见且滤波器未初始化时，FOV 类算法返回零加速度，并让机头指向前方搜索点。
- 丢失目标时导引增益乘以 `lost_guidance_gain=0.55`，避免基于旧信息过度机动。
- 该机制模拟视觉追踪中目标短时遮挡或离开画面后的搜索行为。

## 7. 对比算法设置

本节建议先列表，再分段介绍。重点算法是 `PN + FOV + NMPC`。

| 算法名称 | 论文中角色 | 主要功能 |
| --- | --- | --- |
| Direct pursuit | 基线算法 | 速度方向始终指向真实目标当前位置 |
| Direct pursuit + FOV | 视场约束基线 | 使用深度相机量测/滤波估计目标，再执行直接追踪 |
| PN + FOV | 视场约束比例导引 | 使用深度相机量测/滤波估计目标，再执行三维比例导引 |
| PN + FOV + CBF | 安全滤波对比 | 接近 FOV 边界时加入侧向修正 |
| PN + FOV + MPPI | 采样预测控制对比 | 随机采样控制序列并按代价加权 |
| PN + FOV + NMPC | 本文重点算法 | 在 PN 趋势附近滚动优化候选控制 |

### 7.1 Direct pursuit

可写要点：

- 计算追踪机到目标当前位置的单位方向。
- 设定沿该方向的期望巡航速度。
- 通过一阶速度跟踪得到加速度指令。
- 优点是简单稳定，缺点是不能预测目标未来运动，对横向运动目标拦截效率较低。

### 7.2 Direct pursuit + FOV

可写要点：

- 在 Direct pursuit 基础上加入有限视场判断和传感器估计约束。
- 目标可见时使用深度相机位置量测经 α-β 滤波得到的目标参考。
- 目标不可见时使用滤波器常速度预测，并降低导引增益。
- 该算法用于分离“直接追踪策略”和“有限视场传感器约束”的影响。

### 7.3 PN + FOV

可写要点：

- 在 PN 基础上加入有限视场判断。
- 目标可见时使用深度相机位置量测和 α-β 滤波得到的目标参考进行 PN 导引。
- 目标丢失时使用滤波器常速度预测点，并降低导引增益。
- 该算法是后续 CBF、MPPI、NMPC 的共同基线。

### 7.4 PN + FOV + CBF

可写要点：

- CBF 版本并不替代 PN，而是在 PN 名义控制上叠加 FOV 安全修正。
- 当 LOS 角接近半视场角边界时，算法沿横向误差方向加入加速度修正。
- 在边界附近混入速度匹配项，减少近距离飞越后目标被甩出视场的概率。
- 若一步预测发现修正后 FOV 评分反而变差，则回退到原 PN + FOV 控制。

### 7.5 PN + FOV + MPPI

可写要点：

- MPPI 以 PN 加速度为名义控制序列。
- 在预测窗口内采样多条带噪声的加速度序列。
- 对每条序列进行批量前向仿真，计算距离、FOV 违反、控制能量、平滑性和偏离 PN 趋势等代价。
- 根据代价指数权重融合第一步控制。
- 该方法通常能明显降低 FOV 丢失时长，但计算量高于 PN/CBF。

### 7.6 PN + FOV + NMPC（重点）

本节是论文仿真部分的重点，建议写得最详细。

可写要点：

- NMPC 不是完全替代 PN，而是作为 PN + FOV 外层的滚动预测控制增强。
- 算法首先计算 PN 名义控制 `a_PN`。
- 然后构造一组候选加速度，包括原始 PN、缩放 PN、混入直接拦截项、同速接近项、速度匹配项以及 LOS 垂直方向扰动项。
- 对每个候选加速度，在预测窗口内使用相同追踪机动力学前向滚动。
- 预测目标位置时仅使用当前参考目标位置和速度进行匀速外推，避免使用真实未来轨迹。
- 每个候选根据综合代价函数评分，选择代价最低的候选作为当前控制。
- 只执行当前一步控制，下一控制周期重新求解，形成滚动优化。

建议强调：

- 该 NMPC 是轻量 shooting-based candidate optimization，不依赖 CasADi/acados 等外部优化求解器。
- 它更适合本项目的实时仿真和 Gazebo 接入需求。
- 与 MPPI 的随机采样不同，NMPC 候选由导引几何和任务意图构造，可解释性更强。
- 与 CBF 相比，NMPC 不是只在 FOV 边界局部修正，而是在预测窗口内同时考虑捕获、视场和控制平滑。

## 8. NMPC 代价函数设计

本节建议作为重点算法的方法解释，也可以放在算法章节之后、结果章节之前。

可写要点：

- 代价函数包含终端距离、预测窗口路径距离、FOV 违反、控制能量、控制平滑性和偏离 PN 趋势惩罚。
- 终端距离用于保证最终接近目标。
- 路径距离用于鼓励整个预测窗口内持续接近目标。
- FOV 违反项用于约束 LOS 角不超过半视场角。
- 控制能量项用于避免过大的加速度消耗。
- 平滑项用于降低相邻控制指令突变。
- PN 趋势项用于避免 NMPC 选择完全背离经典导引方向的异常控制。

建议公式形式：

```text
J = w_d J_terminal + w_p J_path + w_f J_fov + w_u J_control + w_s J_smooth + w_n J_pn
```

默认权重：

| 权重 | 数值 | 含义 |
| --- | ---: | --- |
| `nmpc_w_dist` | 12.0 | 终端距离权重 |
| `nmpc_w_path` | 0.1 | 路径距离权重 |
| `nmpc_w_fov` | 10.0 | FOV 违反权重 |
| `nmpc_w_control` | 0.015 | 控制能量权重 |
| `nmpc_w_smooth` | 0.08 | 控制平滑权重 |
| `nmpc_w_pn` | 0.04 | 偏离 PN 趋势权重 |

预测窗口参数：

| 参数 | 数值 | 含义 |
| --- | ---: | --- |
| `horizon_steps` | 20 | 预测步数 |
| `mpc_dt` | 0.1 s | 预测步长 |
| 预测时域 | 2.0 s | `horizon_steps * mpc_dt` |

## 9. 性能评价指标

本节应说明每个指标的物理意义，后续结果分析直接引用。

| 指标 | 含义 | 评价方向 |
| --- | --- | --- |
| `capture_time` | 第一次进入捕获半径的时间，未捕获记为 40 s | 越小越好 |
| `min_distance` | 仿真过程中的最小相对距离 | 越小越好 |
| `mean_distance` | 平均相对距离 | 越小越好 |
| `path_length` | 追踪机轨迹长度 | 越短通常越经济 |
| `control_energy` | `sum(||a||^2 dt)` | 越小表示控制更省 |
| `lost_duration` | 目标不在视场内的累计时间 | 越小越好 |
| `visible_ratio` | 目标可见时间占比 | 越大越好 |
| `yaw_rate_mean` | 平均 yaw 角速度绝对值 | 越小越平滑 |
| `yaw_rate_variance` | yaw 角速度方差 | 越小越平滑 |

建议说明：

- 对无 FOV 约束的 Direct pursuit，代码按理想传感器处理，`visible_ratio=1`、`lost_duration=0`，因此它的视场指标不能直接与 FOV 算法解释为同一传感器条件下的真实可见性。
- 对 `Direct pursuit + FOV`、`PN + FOV`、`CBF`、`MPPI`、`NMPC`，视场指标反映有限视场追踪能力。

## 10. 离线 Python 仿真流程

本节说明如何生成 `outputs/stationary`、`outputs/linear`、`outputs/circle` 中的结果。

可写要点：

- 使用 `main.py` 运行三种场景下的全部算法。
- 每个场景中，六种算法使用相同追踪机初始状态、相同目标轨迹和相同约束。
- 仿真循环先根据当前时刻生成目标状态，再计算导引加速度，随后记录当前状态并积分到下一步。
- 输出包括 `metrics.csv`、`trajectory_3d.png`、`trajectory_xy.png`、`trajectory_xz.png`、`distance_error.png`、`fov_visibility.png`、`acceleration.png`、`yaw_rate.png`、`metrics.png` 和 `replay.mcap`。

建议给出运行命令：

```bash
cd 6_Simulation
uv run python main.py --scenario all --export-mcap
```

### 11.4 离线仿真综合结论

建议总结：

- Direct pursuit 简单稳定，但面对运动目标尤其是圆周目标时捕获效率较低，且属于理想传感器基线。
- Direct pursuit + FOV 加入有限视场量测和滤波后仍可捕获，但容易长时间失锁。
- PN + FOV 引入有限视场量测/滤波机制后，暴露出单纯 PN 在有限视场下容易长时间失锁的问题。
- CBF 能一定程度降低 FOV 丢失，但对复杂机动目标的改善有限。
- MPPI 能显著提高可见率，但控制能量和 yaw rate 平滑性不如 NMPC。
- `PN + FOV + NMPC` 在三种离线场景中均表现出低控制能量、低 yaw rate 和较高视场保持能力，是本文方法的主要优势。

## 12. ROS2/PX4/Gazebo 闭环仿真设计

本节说明 Gazebo 验证不是重新实现算法，而是复用离线算法并接入 PX4 控制链路。

### 12.1 双机仿真结构

可写要点：

- 系统包含一架追踪机和一架目标机。
- 目标机按照合成目标轨迹飞行。
- 追踪机通过 PX4 Offboard 模式接收速度与加速度 setpoint。
- 目标机和追踪机均由 PX4 SITL 管理底层控制。
- ROS2 节点只负责导引计算、坐标转换、setpoint 发布和数据记录。

### 12.2 ROS2 话题与 PX4 消息

订阅：

- `/<pursuer_namespace>/fmu/out/vehicle_odometry`
- `/<target_namespace>/fmu/out/vehicle_odometry`

发布：

- `/<pursuer_namespace>/fmu/in/offboard_control_mode`
- `/<pursuer_namespace>/fmu/in/trajectory_setpoint`
- `/<pursuer_namespace>/fmu/in/vehicle_command`
- `/<target_namespace>/fmu/in/offboard_control_mode`
- `/<target_namespace>/fmu/in/trajectory_setpoint`
- `/<target_namespace>/fmu/in/vehicle_command`

可写要点：

- `OffboardControlMode` 用于告诉 PX4 当前外部控制启用位置、速度或加速度字段。
- `TrajectorySetpoint` 用于发送位置、速度、加速度和 yaw 指令。
- `VehicleCommand` 用于发送 arm 和切换 Offboard 模式命令。
- `VehicleOdometry` 提供追踪机和目标机的实际位置、速度与姿态信息。

### 12.3 启动与控制流程

可写要点：

- 节点等待两架机均发布有效 `VehicleOdometry` 后再开始控制。
- 首先控制目标机进入 Offboard，并飞到当前场景的初始目标点。
- 目标机到达并基本停稳后，追踪机才开始 Offboard 预热、解锁和准备。
- 两机准备完成后，仿真计时开始。
- 目标机持续发布理想轨迹 setpoint。
- 追踪机每个控制周期读取当前目标机真实 odometry，调用 `compute_guidance()` 计算导引加速度。
- 追踪机将加速度指令积分为速度 setpoint，并同时发布速度与加速度控制。

### 12.4 Gazebo 数据记录

可写要点：

- 节点退出时保存 `gazebo_samples.csv`。
- CSV 字段包括追踪机位置速度、目标机位置速度、加速度指令、yaw、pitch、距离、visible 和 LOS 角。
- 后处理脚本 `plot_gazebo_csv.py` 将 CSV 读回 `SimulationResult`，复用离线仿真的指标计算和绘图函数。
- 这样可以保证 Gazebo 闭环结果与离线仿真结果采用相同评价指标。

## 13. Gazebo 闭环仿真结果分析提纲

当前 `outputs/gazebo/circle/` 中已有圆周目标场景结果，包含 Direct pursuit、Direct pursuit + FOV、PN + FOV、PN + FOV + CBF、PN + FOV + MPPI、PN + FOV + NMPC，可用于与离线仿真采用相同指标进行对比。

- Direct pursuit 在 Gazebo 圆周场景下捕获时间为 12.95 s，慢于离线质点仿真，说明真实飞控闭环引入了额外动态滞后。

## 15. 仿真假设与局限性

本节建议放在仿真章节结尾，主动说明模型边界。

可写要点：

- 离线仿真采用三维质点模型，不包含完整四旋翼姿态、推力、电机和气动模型。
- yaw/pitch 主要用于有限视场判断，不是完整六自由度姿态动力学。
- FOV 可见性只考虑几何夹角，没有模拟图像噪声、遮挡、检测延迟和误检漏检。
- 目标丢失后的预测采用匀速模型，不能覆盖目标强机动或复杂行为预测。
- Gazebo 部分验证了 ROS2/PX4 Offboard 闭环接入，但结果受 PX4 控制器参数、机体模型、目标机跟踪误差和通信频率影响。
- 当前 Gazebo 圆周场景中 NMPC 更偏向视场保持，后续可通过重新整定权重提高最终捕获裕度。
